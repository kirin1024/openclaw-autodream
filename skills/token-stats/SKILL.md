# token-stats
Token 消耗统计 — 消息末端标准格式

## 触发条件
每次对话回复时自动执行。

## 规则内容

所有 agent（main 和 subagent）在发送消息到飞书群时，必须在消息**末尾**附加 Token 统计行：

```
[📊 Tokens: {输入} in / {输出} out | 模型: {模型名}]
```

## 实现步骤

### 第1步：获取当前会话状态
调用 `session_status` 获取 tokens 和 model 字段。

### 第2步：提取 Token 数据
tokens 字段格式为 `{输入} in / {输出} out`（本轮增量，非累计）。

### 第3步：附加到消息末尾
在回复内容后换行，追加统计行。

## 示例

**原始回复：**
```
今天的持仓情况如下：
- 紫金矿业：33.5元，浮亏 2.7%
- 白银：3.2元，浮亏 11%

暂无操作建议。
```

**加上 Token 统计后：**
```
今天的持仓情况如下：
- 紫金矿业：33.5元，浮亏 2.7%
- 白银：3.2元，浮亏 11%

暂无操作建议。

[📊 Tokens: 120 in / 85 out | 模型: minimax-portal/MiniMax-M2.7]
```

## 注意事项
- session_status 返回的 token 精度有限，为估算值
- 如为首次对话（无历史），显示累计值即可
- Token 统计行与正文之间留一个空行

## 为什么要有这条规则
1. 方便追踪每个对话的消耗，控制成本
2. 方便复盘某个任务或 agent 的总消耗
3. 统一格式便于日志分析

## 适用对象
- **main agent**：自己处理的消息，直接追加
- **sub agent**：返回给 main 的内容，由 main 统一追加后再发送
